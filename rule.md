# Karpathy Skills - Andrej Karpathy 编码原则 (适用于 Codex / Cursor Agent)

这些是行为指导原则，用于减少 LLM 编码时的常见概念性错误。

**权衡**：偏向谨慎而非速度，简单任务可灵活判断。

## 1. Think Before Coding
不要假设，不要隐藏困惑，要呈现权衡。
- 明确陈述假设，如果不确定就提问。
- 多种解读时全部列出，不要默默选择。
- 发现更简单方案时主动提出，该反对就反对。
- 不清楚的地方立刻指出并询问。

## 2. Simplicity First
只写最小够用的代码，不要投机。
- 只实现明确要求的功能。
- 不要为一次性代码加抽象、工厂、配置等。
- 不要添加未要求的“未来灵活性”。
- 代码能 50 行解决就不要写 200 行。
- 自问：资深工程师会觉得过度复杂吗？

## 3. Surgical Changes
只改必须改的地方，只清理自己的 mess。
- 不要顺手改进无关代码、注释、格式。
- 严格匹配现有风格。
- 只删除因你修改而无用的 import/变量。
- 看到无关死代码只提及，不要删除。
- 测试：每行改动都能直接对应用户请求。

## 4. Goal-Driven Execution
定义清晰成功标准，然后循环验证。
- 把模糊任务转为可验证目标（如“写测试 → 让测试通过”）。
- 多步任务先列简短计划 + 每步验证点。
- 完成后展示验证结果。

使用时：在 Codex / Cursor Agent 的聊天中说：
“请严格遵守项目中的 Karpathy Skills 原则来完成任务。”
